本模組負責將大量保險條款 PDF 檔案，高效且完整地轉換為純文字格式，為後續的 RAG（檢索增強生成）系統之切片（Chunking）與向量化（Embedding）提供乾淨的文本輸入。

## 📊 專案開發紀錄 (STAR)

### **Situation (情境與背景)**
自保發中心（tii）大量爬取了 1,542 份健康保險的保單條款 PDF 檔案。由於保險條款內含大量複雜表格、費率表及非結構化文字，如何完整且結構化地萃取文字成為 RAG 系統精準度的關鍵。

### **Task (任務)**
將 `datasets/保單條款.zip` 壓縮包內的所有 PDF 檔案，批量轉換為純文字格式，並輸出至 `tii_txts/` 目錄中。要求支援斷點續跑（自動跳過已轉換檔案），且必須符合指導老師指定使用 `docling` 框架的要求。

### **Action (開發行動與技術嘗試)**

在中間開發過程中，我們針對本機筆電效能瓶頸與專題機的離線環境，進行了三次主要的架構調整：

1. **嘗試一：初版 `convert_pdfs.py` (純本地端 CPU 運轉)**
* **做法**：逐一讀取本地資料夾的 PDF，使用 `docling` 預設模型（開啟 LayoutLM 與 Tableformer）進行深度結構解析。
* **遇到問題**：保險條款表格極度密集，純 CPU 運算導致速度極慢（每份平均耗時 14 分鐘），全批量預估需不間斷執行 15 天。第 20 頁後的費率大表更引發底層 C++ 引擎記憶體溢位，噴出 `std::bad_alloc` 錯誤。


2. **嘗試二：優化版 `convert_pdfs_pypdf.py` (免解壓直讀 + 核心關閉)**
* **做法**：改為利用 `zipfile` 在記憶體中直讀 ZIP 串流，避免硬碟 I/O 損耗；並在 `docling` 初始化時強制設定 `do_ocr=False` 與 `do_table_structure=False`，關閉耗時的表格與佈局辨識模型。
* **遇到問題**：速度雖縮短至每份 54 秒，但搬遷至內部網路（離線斷網）的專題機執行時，舊版 `docling` 因斷網無法向 Hugging Face 下載純文字版面模型，拋出 `Cannot find cached snapshot` 錯誤；同時因權限隔離卡死，噴出 `.cache/huggingface/token Permission denied` 權限拒絕，導致全數轉換失敗。


3. **嘗試三：終極版 `(for demo-notebook)convert_pdfs_v.1.py` (環境繞過 + 雙重混合轉換引擎)**
* **做法**：在程式碼頂端注入環境變數 `HF_HOME="/tmp/katy_hf_home"` 與 `HF_HUB_OFFLINE=1`，強制快取改道並停用聯網檢查，繞過專題機權限與斷網限制。同時，設計 **Fallback 機制**：優先使用 `docling` 解析結構，若偵測到離線缺乏模型或權限報錯，程式不崩潰，自動無縫切換至 `pypdf` 引擎提取純文字頂替。



### **Result (成果)**

* **執行時間**：整體轉換速度從原先的每份 14 分鐘，提升至 **每秒 5.28 份**，1,542 份檔案在專題機上僅花費 **6 分 11 秒** 即全數處理完畢。
* **成功率**：最終達到 **1,540 份成功轉換**。其餘 2 份失敗經 Log 稽核確認為原始 ZIP 內即毀損之 0 KB 空檔。
* **架構優勢**：將輸入（`datasets/` 原始唯讀資料）與輸出（`tii_txts/` 清洗後文字）明確分離。資料夾獨立乾淨，防止 Raw Data 被污染，並便於下一步 `chunking_langchain_testbed.ipynb` 的 glob 批量讀取。

## 🛠️ Python 腳本版本差異比較

專案資料夾內包含兩個主要版本的轉換程式，其核心技術差異如下：

| 比較項目 | `convert_pdfs_pypdf.py` (本機優化版) | `convert_pdfs_v.1.py` (專題機混合相容版) |
| --- | --- | --- |
| **讀取對象** | 本地端解壓後的 PDF 實體檔案資料夾。 | 直接讀取相對路徑的 `保單條款.zip` 壓縮包。 |
| **路徑寫法** | Windows 絕對路徑（寫死於本機電腦）。 | 動態相對路徑（自動抓取腳本所在目錄，具可攜性）。 |
| **並發處理** | 單執行緒依序排隊處理。 | 移除了舊版不相容的 `num_threads`，採作業系統級 I/O 優化。 |
| **環境適應力** | 僅能在具備網路、擁有完全讀寫權限的個人筆電執行。 | 自帶 `HF_HOME` 權限繞過與快取改道，適應**離線斷網、無 sudo 權限**的專題機。 |
| **容錯機制** | 遇到解析失敗或環境缺件時，程式會拋出 Exception 停止。 | **具備 Fallback 機制**。`docling` 斷網出錯時，自動切換為 `pypdf` 提取純文字。 |
