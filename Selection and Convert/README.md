目前問題是篩選未貼合使用者的提問、重複率太高

1. 條款內文根本不會寫「智齒」或「粉瘤」保險公司的保單條款屬於嚴謹的法律與醫學文件，內文只會寫「門診手術保險金」、「住院醫療費用」或「除外責任」，絕對不會把「智齒切除」或「粉瘤」等具體病名寫進常規條款裡。
因此，腳本在篩選時，只能用大方向的法律與理賠關鍵字（如：實支實付、住院日額、特定傷病）去盲測 PDF 的前兩頁。它是依據「這張保單有沒有包含這類理賠條款」來選，而不是依據「有沒有這兩個字」來選。

2. 腳本的關鍵字規則過於機械化如果我們去看 select_and_convert.py 的原始碼設定：PythonCATEGORIES = {
    "一、醫療險理賠實務與條款認定": ["實支實付", "住院費用", "手術費用", ...],
    "二、醫療與健康險保單健檢及預算規劃": ["住院日額", "日額型", "定額給付", ...],
    ...
}
這個分類器非常單純，它只檢查 PDF 檔名和前兩頁有沒有出現這些特定字眼。
當它掃描到 三商美邦人壽團體365住院醫療保險 或 114學年度學生團體保險 時，因為這些檔案的檔名或開頭剛好大量重複出現 住院醫療、癌症 等字眼，導致它們在分數上「非常高分」，在每個分類裡都名列前茅。加上腳本寫死了 MAX_PER_CATEGORY = 3（每類只取前 3 份），後面真正多元的保單就直接被截斷丟棄了。

3. 被選中的保單，其實已經涵蓋了那些常見問題雖然這 15 份保單的檔名看起來很枯燥，但從理賠角度來看，它們完全貼合你列出的常見問題。例如：篩選出來的 三商美邦團體住院/手術險，裡面保障的「門診手術」與「雜費認定」條款，正是用來判定「智齒複雜切除術」與「粉瘤手術」能不能理賠的核心依據。篩選出來的 全球人壽醫卡讚/防癌險，裡面的條款正是使用者在詢問「全球保單附加癌險」與「重大傷病 XDE 比較」時，知識庫必須撈出來比對的標準答案。
